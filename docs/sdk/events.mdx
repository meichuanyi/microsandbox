## Sandbox methods

### on_event(event_name, callback) / onEvent(eventName, callback)

Subscribe to named events from the guest. The callback receives `EventData` each time a matching event arrives. Returns a `Subscription` handle.

### emit(event_name, data)

Send a named event with typed data to the guest. Any process listening on the agent socket receives it.

---

## EventData

- `event: String` — event name
- `data: Value` — event payload (JSON-compatible)

## Guest-side protocol

Write newline-delimited JSON to `/run/agent.sock` (Unix domain socket):

```json
{"event": "task.progress", "data": {"percent": 50, "message": "Halfway done"}}
```

No SDK required on the guest side.
