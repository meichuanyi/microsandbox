## Static methods

### Sandbox.builder(name) / Sandbox.create(config)

Create and boot a sandbox. The sandbox is "attached" by default ‚Äî it stops when your process exits.

### Sandbox.create_detached(config) / Sandbox.createDetached(config)

Create and boot a sandbox that survives after your process exits.

### Sandbox.start(name)

Restart a stopped sandbox from persisted state. Returns `Sandbox`.

### Sandbox.get(name)

Get a handle to an existing sandbox. Returns `SandboxHandle`.

### Sandbox.list()

List all sandboxes. Returns `Vec<SandboxInfo>`.

### Sandbox.remove(name)

Delete a stopped sandbox and its state from disk.

## Instance methods

### exec(cmd, args?)

Run a command and collect output. Returns `ExecOutput`. See [Execution](/sdk/execution).

### shell(script)

Run through the sandbox's configured shell. Returns `ExecOutput`.

### fs()

Get a filesystem handle. Returns `SandboxFs`. See [Filesystem](/sdk/filesystem).

### metrics()

Get point-in-time resource metrics. Returns `SandboxMetrics`. See [Metrics](/sdk/metrics).

### metrics_stream(interval_ms) / metricsStream(intervalMs)

Stream metrics at a regular interval. Returns `MetricsStream`.

### stop()

Graceful shutdown (SIGTERM).

### stop_and_wait() / stopAndWait()

Stop and wait for exit. Returns `ExitStatus`.

### kill()

Force terminate (SIGKILL).

### pause() <sup>coming soon</sup>

Freeze all guest processes. Zero CPU while paused.

### resume() <sup>coming soon</sup>

Resume from pause. Instant, no re-boot.

### drain()

Graceful drain ‚Äî finish existing commands, reject new ones.

### detach()

Release the handle. Sandbox continues running as a background process.

### wait()

Block until the sandbox exits on its own. Returns `ExitStatus`.

### snapshot() <sup>coming soon</sup>

Capture full VM state. Returns `Snapshot`. See [Snapshots](/sdk/snapshots).

### on_event(name, callback) <sup>coming soon</sup>

Subscribe to guest events. See [Events](/sdk/events).

### emit(name, data) <sup>coming soon</sup>

Send event to guest. See [Events](/sdk/events).

## SandboxBuilder / SandboxConfig

### Core options

- `image: String` ‚Äî OCI image, local path, or disk image (required)
- `cpus: u32` ‚Äî virtual CPUs (default: `1`)
- `memory / memoryMib: u32` ‚Äî guest memory in MiB (default: `512`)
- `workdir: String` ‚Äî default working directory
- `shell: String` ‚Äî shell for `shell()` calls (default: `/bin/sh`)
- `env: Map<String, String>` ‚Äî environment variables
- `replace: bool` ‚Äî replace existing sandbox with same name (default: `false`)

### Volumes

- `volume(path, config) / volumes: Map` ‚Äî bind, named, or tmpfs mounts. See [Volumes](/sdk/volumes).

### Network

- `network(config) / network: NetworkPolicy` ‚Äî network policy and configuration. See [Networking](/sdk/networking).
- `port(host, guest)` ‚Äî publish a TCP port
- `port_udp(host, guest)` ‚Äî publish a UDP port

### Secrets

- `secret(config)` ‚Äî add a secret with full configuration. See [Secrets](/sdk/secrets).
- `secret_env(env, value, host)` ‚Äî shorthand for header-injected secret

### Policies

- `max_duration / maxDurationSecs: u64` ‚Äî maximum lifetime in seconds
- `idle_timeout: u64` ‚Äî auto-drain after idle seconds
- `log_level: LogLevel` ‚Äî override log verbosity

### Image options

- `image_with(config)` ‚Äî disk image with explicit filesystem type
- `registry_auth: RegistryAuth` ‚Äî credentials for private registries
- `pull_policy: PullPolicy` ‚Äî `Always`, `IfMissing` (default), `Never`

## Types

### SandboxInfo

- `name: String` ‚Äî sandbox name
- `status: String` ‚Äî `Running`, `Stopped`, `Crashed`, etc.

### ExitStatus

- `code: i32` ‚Äî exit code
- `success: bool` ‚Äî `true` if exit code is 0

### SandboxMetrics

- `cpuPercent: f64` ‚Äî CPU usage percentage
- `memoryBytes: u64` ‚Äî current memory usage
- `memoryLimitBytes: u64` ‚Äî memory limit
- `diskReadBytes: u64` ‚Äî total disk bytes read
- `diskWriteBytes: u64` ‚Äî total disk bytes written
- `netRxBytes: u64` ‚Äî total network bytes received
- `netTxBytes: u64` ‚Äî total network bytes sent
- `uptimeMs: u64` ‚Äî sandbox uptime in milliseconds
- `timestampMs: u64` ‚Äî measurement timestamp
